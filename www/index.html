Allowlist

If at least one allow rule applies to a host, ONLY listed IPs/ranges are allowed through that host. Leave empty (or scope to other hosts) to disable allowlisting elsewhere.

IP or CIDRHosts (glob, comma-sep; empty = all)

Blocklist

IP or CIDRHosts (glob, comma-sep; empty = all)

User-Agent blocklist

Regex (case-insensitive prefix: (?i)). Matched against the request's User-Agent header. Except paths (comma-sep substrings) skip the rule when the request URI contains any of them — useful for letting bots fetch /robots.txt while blocking everything else.

PatternHostsExcept paths

WAF rules

Go regex, matched against request URI, full URL, Cookie, and Referer.

OnNamePatternHosts

Per-IP rate limit

Limits each source IP globally across all hosts the plugin is assigned to.

Honeypot paths

Any request whose URI matches one of these patterns triggers an immediate temporary ban on the source IP. Use literal paths (e.g. /.env) for prefix-substring matches, or include * for glob patterns. Bots probe these constantly — a perfect signal source.

PathHosts (glob, comma-sep; empty = all)

Auto-ban escalation

After an IP triggers any block rule (UA, WAF, rate-limit, etc.) Threshold times within Window seconds, it's added to the temp-ban list for Ban duration. Honeypot hits already install temp bans directly.

Host header blocklist

Regex matched against the request's Host header. Useful for rejecting traffic where someone is probing your Zoraxy IP with a bogus host (e.g. Host: localhost).

RegexHosts (apply scope; usually leave empty)

Import Cloudflare rule

Paste a Cloudflare WAF / Custom Rule expression. Guardian translates the predicates it understands into honeypot paths, UA rules, host-header blocks, and WAF rules. Anything it can't translate (e.g. cf.threat_score, GeoIP, TLS fingerprints) is listed as a warning. Click Preview first, then Apply to merge into your live config.

Enable when Guardian is behind Zoraxy (the normal case). Disable only if you don't trust the upstream chain.

    
Host pattern syntax

    

      example.com — exact match.

      *.example.com — single subdomain level (does not match a.b.example.com).

      **.example.com — any subdomain depth.

      * or empty — any host.
